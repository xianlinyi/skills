# Commit Message Template

Use this structure unless the user requested another convention.

<type>(<scope>): <subject>

<body>

<footer>

Guidelines:
- type: feat, fix, chore, docs, refactor, test, perf, ci, build.
- subject: imperative, <= 72 chars.
- body: explain why and what changed.
- footer: include breaking changes or issue links when applicable.
